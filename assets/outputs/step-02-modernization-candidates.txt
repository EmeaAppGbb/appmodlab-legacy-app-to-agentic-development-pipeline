6. PATTERNS IDENTIFIED FOR AGENT EXPLOITATION
────────────────────────────────────────────────────────────────────────────────

Pattern 1: State-Specific Rule Variations
  Files: QuotingEngine.cs, ComplianceService.cs, ComplianceRules.cs
  Pattern: switch/if-else on StateCode with unique logic per state
  Agent Approach: Generate per-state configuration + rule engine

Pattern 2: Rating Factor Application
  Files: QuotingEngine.cs, RatingRules.cs
  Pattern: Lookup factor → multiply against base → cap/floor result
  Agent Approach: Data-driven factor tables + generic applicator

Pattern 3: Risk Evaluation Components
  Files: UnderwritingService.cs, UnderwritingRules.cs
  Pattern: Evaluate aspect → score → accumulate → decide
  Agent Approach: Pluggable evaluator interface + scoring engine

Pattern 4: CRUD + Lifecycle Management
  Files: PolicyService.cs, EndorsementService.cs, RenewalService.cs
  Pattern: Create → validate → persist → transition state
  Agent Approach: Generic repository + state machine generator

Pattern 5: External Integration Adapters
  Files: ReinsuranceClient.cs, RegulatoryReporter.cs
  Pattern: Build request → call external → parse response → handle errors
  Agent Approach: HTTP client generator + Polly policies + mock services
