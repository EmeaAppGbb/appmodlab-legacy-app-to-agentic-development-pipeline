## 2. Key Complexity Areas

### QuotingEngine.cs ÔÇö **687 lines** (Highest complexity)
- **17-step premium calculation algorithm** with 40+ rating factors across 50 states
- Hardcoded lookup dictionaries for base rates (20 states), occupancy factors (11 types), territory factors, catastrophe zones, state taxes, and minimum premiums
- Factors: property value (7 tiers), construction type (6 classes), building age (8 bands), occupancy (11 types + state adjustments), protection (sprinklers/alarm with combined bonus), territory/ZIP, catastrophe zone (hurricane/earthquake/tornado), roof type (6 types) ├ù age (5 bands), square footage (6 tiers), stories (6 bands), loss history (frequency ├ù severity), deductible credit (6 tiers)
- Optional coverage calculations: Business Interruption, Equipment Breakdown, Flood, Earthquake
- State-specific surcharges/taxes + TRIA terrorism surcharge + fire marshal fees
- **No dependency injection** ÔÇö instantiates `PremiumCalculator` and `ComplianceService` directly

### UnderwritingService.cs ÔÇö **297 lines**
- Risk scoring (0ÔÇô100 scale) with multi-factor evaluation
- 4-outcome decision tree: Approved / Declined / ReferToSenior / RequestMoreInfo
- Probable Maximum Loss (PML) calculation with state-specific adjustments
- Evaluates 5 risk dimensions: Construction, Occupancy, Protection, Loss History, Catastrophe Exposure
- Auto-decline triggers: risk score > 85 or PML > 50% of property value

### ComplianceService.cs ÔÇö **202 lines**
- State-by-state regulatory validation (CA Prop 103, FL building code, TX wind/hail, NY replacement cost, LA hurricane deductible)
- Per-state compliance rules with warnings and hard blocks

### PolicyService.cs ÔÇö **150 lines**
- Full policy lifecycle: issue, cancel (pro-rata/short-rate/flat), renew
- Reinsurance cession logic (>$2M property value ÔåÆ 60% ceded)

### EndorsementService.cs ÔÇö **133 lines**
- Mid-term policy modifications: coverage limit changes, deductible changes, cancellations
- Pro-rated premium adjustments with deductible credit tables

### RenewalService.cs ÔÇö **107 lines**
- Automated renewal quote generation with inflation (3%), state-specific trend factors (3ÔÇô12%)
- Re-invokes full QuotingEngine for premium recalculation

## 7. Key Risks & Anti-Patterns

1. **No dependency injection** ÔÇö All services use `new` to create dependencies (untestable)
2. **Hardcoded business rules** ÔÇö Rate tables, state factors, and thresholds embedded in C# code instead of database/configuration
3. **Dual premium calculation** ÔÇö Both C# (`QuotingEngine`) and SQL (`usp_CalculatePremium`) compute premiums ÔåÆ drift risk
4. **Magic strings everywhere** ÔÇö Status values, construction types, occupancy types as raw strings (no enums)
5. **DateTime.Now coupling** ÔÇö Timestamps hardcoded, making testing difficult
6. **God class** ÔÇö `QuotingEngine` (687 lines) handles all 17 calculation steps in one class
7. **No error handling** ÔÇö Services lack try/catch; failures propagate as unhandled exceptions
8. **Monolithic data model** ÔÇö Quote entity has 30+ properties spanning property, coverage, and claims concerns
